## 3. 功能模組操作指南

### 3.1 客戶管理模組

**路徑**：`/clinic/customers`

**功能說明**：管理診所所有客戶資料，包含基本資料、會員等級、消費記錄、標籤分類等。

#### 操作步驟

**新增客戶**：
1. 點擊頁面右上角「新增客戶」按鈕
2. 填寫基本資料（姓名、電話、Email、生日）
3. 選擇會員等級（銅牌/白銀/黃金/白金/鑽石）
4. 設定客戶來源（LINE/網站/轉介/廣告等）
5. 點擊「儲存」完成新增

**搜尋與篩選**：
- 使用搜尋框輸入姓名或電話快速查找
- 使用會員等級下拉選單篩選特定等級客戶
- 點擊表頭可依該欄位排序

**客戶詳情**：
- 點擊客戶姓名進入詳情頁面
- 查看消費記錄、療程歷史、預約記錄
- 管理客戶標籤（VIP、敏感肌、回購客等）

#### 相關 API

| 操作 | API 端點 | 說明 |
|------|----------|------|
| 列表 | `trpc.customer.list` | 分頁查詢客戶列表 |
| 詳情 | `trpc.customer.getById` | 取得單一客戶詳情 |
| 新增 | `trpc.customer.create` | 建立新客戶 |
| 更新 | `trpc.customer.update` | 更新客戶資料 |
| 刪除 | `trpc.customer.delete` | 刪除客戶 |

### 3.2 預約管理模組

**路徑**：`/clinic/appointments`

**功能說明**：管理診所預約排程，支援日曆視圖與列表視圖，追蹤預約狀態。

#### 預約狀態流程

```
pending（待確認）
    │
    ├──確認──▶ confirmed（已確認）
    │              │
    │              ├──到診──▶ completed（已完成）
    │              │
    │              └──爽約──▶ no_show（爽約）
    │
    └──取消──▶ cancelled（已取消）
```

#### 操作步驟

**建立預約**：
1. 點擊「新增預約」按鈕
2. 選擇客戶（可搜尋現有客戶或新增）
3. 選擇服務項目（療程/產品）
4. 選擇執行醫師/美容師
5. 選擇預約日期與時段
6. 填寫備註（可選）
7. 點擊「確認預約」

**管理預約**：
- **確認**：將待確認預約改為已確認
- **完成**：客戶到診後標記為已完成
- **取消**：取消預約並記錄原因
- **爽約**：標記未到診的預約

#### 到診率追蹤

系統自動計算以下指標：
- **到診率** = 已完成 / (已完成 + 爽約) × 100%
- **爽約率** = 爽約 / 總預約數 × 100%
- **取消率** = 已取消 / 總預約數 × 100%

### 3.3 療程記錄模組

**路徑**：`/clinic/treatment-records`

**功能說明**：記錄客戶的療程執行詳情，包含前後對比照片、滿意度評分、醫師備註等。

#### 操作步驟

2. 選擇執行的療程項目
3. 填寫療程詳情（使用劑量、部位等）
4. 上傳療程前後照片（支援多張）
5. 填寫醫師備註與建議
6. 設定客戶滿意度評分（1-5 分）
7. 點擊「儲存記錄」

**照片管理**：
- 支援拖曳上傳多張照片
- 自動分類為「療程前」與「療程後」
- 支援前後對比展示模式

### 3.4 療程套餐管理

**路徑**：`/clinic/customer-packages`

**功能說明**：管理客戶購買的療程套餐，追蹤堂數使用與到期狀態。

#### 套餐狀態

| 狀態 | 說明 |
|------|------|
| `active` | 使用中，尚有剩餘堂數 |
| `expired` | 已過期 |
| `completed` | 堂數已用完 |
| `cancelled` | 已取消/退費 |

#### 操作步驟

**購買套餐**：
1. 選擇客戶
2. 選擇套餐產品
3. 設定總堂數與有效期限
4. 確認付款方式
5. 點擊「確認購買」

**扣除堂數**：
1. 在療程記錄頁面選擇「使用套餐」
2. 選擇要扣除的套餐
3. 系統自動扣除 1 堂並記錄

**到期提醒**：
- 系統自動在到期前 30 天發送提醒
- 可在通知設定中調整提醒天數

### 3.6 RFM 客戶分析

**路徑**：`/clinic/rfm-analysis`

**功能說明**：基於 RFM 模型（Recency 最近消費、Frequency 消費頻率、Monetary 消費金額）進行客戶價值分群分析。

#### RFM 分群定義

| 分群 | R 分數 | F 分數 | M 分數 | 說明 |
|------|:------:|:------:|:------:|------|
| **冠軍客戶** | 4-5 | 4-5 | 4-5 | 最近消費、高頻率、高金額 |
| **忠誠客戶** | 3-5 | 4-5 | 3-5 | 穩定回購的優質客戶 |
| **潛力客戶** | 3-5 | 1-3 | 3-5 | 有消費能力但頻率待提升 |
| **新客戶** | 4-5 | 1 | 1-3 | 剛加入的新客戶 |
| **沉睡客戶** | 2-3 | 2-3 | 2-3 | 曾經活躍但近期減少 |
| **流失風險** | 1-2 | 1-3 | 1-3 | 即將流失的客戶 |
| **已流失** | 1 | 1-2 | 1-2 | 長期未消費的客戶 |

#### 操作步驟

**計算 RFM 分數**：
1. 點擊「重新計算 RFM」按鈕
2. 系統以背景任務方式計算所有客戶的 RFM 分數
3. 進度條顯示計算進度
4. 完成後自動刷新頁面

**查看分析結果**：
- 總覽：各分群客戶數量與佔比
- 高風險客戶警示：流失風險超過 50% 的客戶列表
- 詳細列表：所有客戶的 RFM 分數與分群

**行動建議**：

| 分群 | 建議行動 |
|------|----------|
| 冠軍客戶 | 提供 VIP 專屬優惠、優先預約權 |
| 忠誠客戶 | 推薦新療程、會員升級 |
| 潛力客戶 | 發送回購提醒、限時優惠 |
| 新客戶 | 發送歡迎訊息、首購優惠 |
| 沉睡客戶 | 發送喚醒訊息、專屬折扣 |
| 流失風險 | 電話關懷、大幅優惠 |
| 已流失 | 重新激活活動、問卷調查 |

### 3.8 滿意度調查與 NPS

**路徑**：`/clinic/satisfaction`

**功能說明**：管理客戶滿意度調查，追蹤 NPS（淨推薦值）指標。

#### NPS 計算方式

NPS = 推薦者比例 - 批評者比例

| 評分 | 分類 | 說明 |
|:----:|------|------|
| 9-10 | 推薦者 | 高度滿意，願意推薦 |
| 7-8 | 被動者 | 滿意但不會主動推薦 |
| 0-6 | 批評者 | 不滿意，可能負面口碑 |

#### NPS 分數解讀

| NPS 範圍 | 評價 | 說明 |
|:--------:|------|------|
| 70+ | 卓越 | 極高客戶忠誠度 |
| 50-69 | 優秀 | 客戶滿意度高 |
| 30-49 | 良好 | 有改善空間 |
| 0-29 | 待改善 | 需要重點關注 |
| 負值 | 危險 | 批評者多於推薦者 |

## 4. LINE 整合設定指南

### 4.1 前置準備

在開始設定前，請先完成以下步驟：

1. **建立 LINE Official Account**
   - 前往 [LINE Official Account Manager](https://manager.line.biz/)
   - 建立新的官方帳號（建議選擇「認證帳號」）

2. **啟用 Messaging API**
   - 在官方帳號設定中啟用 Messaging API
   - 記錄 Channel ID 與 Channel Secret

3. **建立 LIFF App**
   - 前往 [LINE Developers Console](https://developers.line.biz/)
   - 在對應的 Channel 下建立 LIFF App
   - 記錄 LIFF ID

### 4.2 LINE Channel 設定

**路徑**：`/clinic/line-settings`

**設定步驟**：

1. **輸入 Channel 憑證**
   - Channel ID：從 LINE Developers Console 取得
   - Channel Secret：從 LINE Developers Console 取得

## 6. 常見問題與故障排除

### 6.1 登入問題

**Q：無法登入系統？**

A：請依序檢查：
1. 確認網路連線正常
2. 清除瀏覽器快取與 Cookie
3. 嘗試使用無痕模式登入
4. 確認帳號權限是否正確設定

### 6.2 LINE 整合問題

**Q：Webhook 沒有收到事件？**

A：請檢查：
1. Webhook URL 是否正確設定
2. 「Use webhook」是否已啟用
3. Channel Access Token 是否有效
4. 伺服器是否正常運作

**Q：Rich Menu 沒有顯示？**

A：請檢查：
1. Rich Menu 是否已發布
2. 用戶是否符合顯示條件
3. 是否有其他 Rich Menu 優先級更高

### 6.3 資料問題

**Q：RFM 分數顯示 NaN？**

A：這表示該客戶尚無消費記錄，系統無法計算 RFM 分數。請先建立療程記錄或訂單。

**Q：庫存數量不正確？**

A：請檢查：
1. 是否有未處理的訂單
2. 是否有手動調整庫存的記錄
3. 查看庫存異動記錄確認變動原因

### 6.4 效能問題

**Q：頁面載入很慢？**

A：建議：
1. 減少單頁顯示的資料筆數
2. 使用篩選條件縮小查詢範圍
3. 避免同時開啟多個大型報表
