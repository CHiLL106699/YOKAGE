# YOChiLL SaaS 平台 Phase 109-114 新功能操作手冊

**版本：** 1.0
**日期：** 2026-02-16

---

## 總覽

本文件旨在說明 YOChiLL SaaS 平台在 Phase 109 至 114 開發週期中所新增的核心功能。這些功能專注於提升 LINE 行銷自動化、數據驅動決策以及客戶關係管理的深度，為您的業務增長提供強大動力。

新增的三大核心功能模組包括：

1.  **Rich Menu 模板市集 (Rich Menu Template Marketplace)**
2.  **A/B 測試 (A/B Testing for Broadcasts)**
3.  **向量搜尋 (Vector Search for Customer Insights)**

以下章節將詳細介紹各功能的操作流程與最佳實踐。

---

## 1. Rich Menu 模板市集

Rich Menu 模板市集讓您無需設計或編碼，即可快速為您的 LINE 官方帳號套用專業、美觀且高互動性的圖文選單。您可以從市集中選擇符合您品牌風格與行銷活動需求的模板，一鍵完成設定。

### 1.1 功能入口

1.  登入 YOChiLL 後台。
2.  於左側導覽列點擊「LINE 渠道」 -> 「Rich Menu 管理」。
3.  點擊頁面右上方的「瀏覽模板市集」按鈕。

### 1.2 操作流程

| 步驟 | 操作 | 說明 |
| :--- | :--- | :--- |
| 1 | **瀏覽與篩選** | 進入市集後，您可以瀏覽所有可用的 Rich Menu 模板。使用頂部的篩選器，可以根據「產業類別」、「色系風格」或「按鈕數量」來快速找到您想要的模板。 |
| 2 | **預覽模板** | 點擊任一模板卡片，即可進入預覽頁面。您可以看到該模板在手機上的實際顯示效果，以及每個按鈕的預設功能。 |
| 3 | **選擇並套用** | 在預覽頁面，點擊「使用此模板」按鈕。系統會將此模板複製一份至您的「Rich Menu 管理」列表中，狀態為「草稿」。 |
| 4 | **編輯與設定** | 回到「Rich Menu 管理」列表，找到剛剛新增的草稿模板，點擊「編輯」。您可以修改每個按鈕的「觸發動作」（例如：開啟網頁、發送訊息、觸發關鍵字回覆等），並填入對應的連結或內容。 |
| 5 | **發布與啟用** | 完成所有按鈕設定後，儲存變更。回到列表，點擊該模板旁的「發布」按鈕，即可將此 Rich Menu 設定為您 LINE 官方帳號的預設選單。 |

### 1.3 最佳實踐

- **定期更換**：根據您的行銷檔期（如：母親節、週年慶）更換不同的 Rich Menu，引導流量至活動頁面。
- **數據追蹤**：搭配平台的點擊追蹤功能，分析哪個按鈕最受歡迎，以優化您的選單設計。

## 2. A/B 測試 (Broadcast A/B Testing)

A/B 測試功能讓您在正式發送大規模推播前，能以科學方法測試不同版本的文案、圖片或優惠，找出成效最佳的組合，從而最大化您的行銷轉換率。

### 2.1 功能入口

此功能整合於「客戶分群推播」模組中。

1.  登入 YOChiLL 後台。
2.  於左側導覽列點擊「客戶分群推播」。
3.  建立一個新的推播活動。

### 2.2 操作流程

1.  **啟用 A/B 測試**：在建立推播活動的頁面，找到「訊息內容」區塊，並勾選「啟用 A/B 測試」選項。

2.  **設定測試版本**：
    - 系統預設會有「版本 A」與「版本 B」兩個分頁。
    - 您可以分別為兩個版本設定不同的「訊息內容」。例如，版本 A 使用較長的優惠說明，版本 B 使用簡潔有力的行動呼籲。
    - 您可以點擊「新增版本」來增加更多測試組（最多支援 4 個版本）。

3.  **設定測試群體**：
    - 在「目標受眾」設定下方，會出現「A/B 測試設定」區塊。
    - **測試群體比例**：您可以設定要用多少比例的目標受眾來進行這次的測試。例如，目標受眾有 10,000 人，您設定測試比例為 20%，則系統會隨機抽出 2,000 人參與測試。
    - **勝出標準**：選擇您衡量成效的指標，可以是「點擊率」或「轉換率」（需搭配轉換追蹤設定）。
    - **測試時長**：設定測試要進行多久（例如：4 小時）。

4.  **執行與監控**：
    - 儲存並排程發送後，系統會先將不同版本的訊息發送給指定比例的測試群體。
    - 您可以在推播活動的詳細頁面中，即時查看各版本的數據表現（發送數、點擊數、點擊率等）。

5.  **自動發送勝出版本**：
    - 測試時長結束後，系統會根據您設定的「勝出標準」，自動選出成效最佳的版本。
    - 該勝出版本的訊息將會自動發送給剩餘的目標受眾（在此範例中為 8,000 人）。

### 2.3 最佳實踐

- **單一變數原則**：每次 A/B 測試盡量只專注於一個變數（例如：只改文案、或只改圖片），這樣才能準確判斷是哪個因素影響了成效。
- **足夠的樣本數**：確保您的測試群體數量足夠大，以避免因偶然性導致的數據偏差。

## 3. 向量搜尋 (Vector Search)

向量搜尋是一項強大的 AI 技術，它能理解語意的相似性，而不僅僅是字面上的關鍵字匹配。在 YOChiLL 平台中，我們將其應用於客戶輪廓分析與互動歷史搜尋，幫助您更精準地找到目標客戶。

### 3.1 功能入口

此功能主要整合於「客戶管理」的全域搜尋框中。

### 3.2 操作範例

傳統的關鍵字搜尋，當您搜尋「修復」時，只會找到包含「修復」二字的客戶標籤或備註。而向量搜尋則能理解更深層的語意。

| 搜尋情境 | 傳統搜尋可能找不到的結果 | 向量搜尋可以找到的結果 |
| :--- | :--- | :--- |
| 搜尋「對價格很敏感的客人」 | 備註為「在意折扣」的客戶 | 備註中包含「覺得太貴」、「詢問有沒有優惠」、「等特價才買」的客戶。 |
| 搜尋「最近想做臉部保養」 | 互動紀錄為「預約了杏仁酸」 | 互動紀錄中包含「詢問皮秒雷射」、「對水飛梭有興趣」、「皮膚最近很乾」的客戶。 |
| 搜尋「高價值潛力客戶」 | 標籤為「VIP」的客戶 | 曾多次詢問高單價療程、消費頻率穩定、或與「已成交大戶」輪廓相似的客戶。 |

### 3.3 如何使用

1.  前往「客戶管理」頁面。
2.  在頁面上方的搜尋框中，直接以「自然語言」輸入您想尋找的客戶特徵或需求。
3.  系統會利用向量搜尋技術，分析所有客戶的標籤、備註、互動歷史、消費紀錄等資訊，並回傳一個按「語意相似度」排序的客戶列表。
4.  相似度分數越高的客戶，代表其輪廓與您的搜尋描述越接近。

### 3.4 最佳實踐

- **描述具體化**：您的描述越詳細具體，搜尋結果就越精準。例如，與其搜尋「想美白的客人」，不如搜尋「詢問過美白針或皮秒雷射，且在意術後反黑問題的客人」。
- **探索新商機**：利用向量搜尋來發掘您從未想過的潛在客戶分群。例如，搜尋「喜歡在下午茶時段來店裡的客人」，可能會找到一群有共同生活型態的客戶，適合推廣專屬的午後方案。
